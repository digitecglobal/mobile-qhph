# Usuario objetivo, personas y JTBD

- Estado: `Borrador`
- Version: `0.1`
- Ultima actualizacion: `2026-02-23`

## 1. Segmentos de usuario

## 1.1 Usuario explorador

Busca planes de forma espontanea para hoy o fin de semana.

## 1.2 Usuario planificador

Organiza con anticipacion y necesita agenda clara, entradas y recordatorios.

## 1.3 Organizador de eventos

Necesita difusion efectiva, publico calificado y conversion a asistencia.

## 2. Personas (resumen)

## Persona A: "Ana, 27, exploradora urbana"

- Objetivo: encontrar planes nuevos cada semana.
- Frustracion: se entera tarde por redes sociales.
- Necesidad: feed corto, relevante y cercano.

## Persona B: "Luis, 33, planificador social"

- Objetivo: coordinar salidas con amigos y pareja.
- Frustracion: falta de info clara (precio, lugar, horarios).
- Necesidad: detalle completo y facil de compartir.

## Persona C: "Colectivo local"

- Objetivo: llenar cupos de sus eventos.
- Frustracion: alcance organico limitado.
- Necesidad: publicar rapido, editar, destacar y medir interes.

## 3. JTBD (Jobs To Be Done)

1. Cuando tengo tiempo libre, quiero descubrir eventos cercanos para elegir rapido que hacer.
2. Cuando me interesa un tipo de plan, quiero recibir alertas antes para no perderlo.
3. Cuando organizo una salida, quiero validar costos, horario y ubicacion en una sola vista.
4. Cuando publico un evento, quiero llegar a personas realmente interesadas.

## 4. User journey actual (dolor)

1. Navegar muchas cuentas/redes.
2. Encontrar eventos desactualizados o incompletos.
3. Llegar tarde a la decision.
4. Friccion para compra o asistencia.

## 5. User journey objetivo (producto)

1. Abrir app y ver eventos relevantes en segundos.
2. Filtrar por fecha/categoria/distancia/precio.
3. Revisar ficha del evento con toda la informacion.
4. Guardar, activar alerta, comprar o compartir.

## 6. Necesidades de accesibilidad y usabilidad

- Texto legible y contraste adecuado.
- Flujo de descubrimiento usable con una mano.
- Soporte para lectura rapida de fecha/hora/lugar.
- Lenguaje claro sin tecnicismos.

## 7. Criterios de prioridad de usuario (MVP)

1. Usuarios exploradores y planificadores urbanos.
2. Organizadores pequenos y medianos de la ciudad piloto.
3. Luego productores mas grandes con integraciones.
