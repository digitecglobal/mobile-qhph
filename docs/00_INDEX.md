## Objetivo

Dejar definido:

- Que problema se resuelve y para quien.
- Que se construye primero (MVP) y que no.
- Como se valida que el producto funciona.
- Como se lanza, se opera y se monetiza.

## Decisiones cerradas (2026-02-24)

- Ciudad piloto inicial: `Bogota, Colombia`.
- Categorias MVP: `musica`, `teatro`, `ferias`, `gastronomia`, `nightlife`.
- Plataforma inicial: `App mobile iOS/Android`.
- Web publica: `Fase posterior, no MVP`.
- Meta 90 dias (basica): `600 WAU`, `120 eventos activos/semana`, `40 organizadores activos mensuales`, `15% retencion semana 4`.
